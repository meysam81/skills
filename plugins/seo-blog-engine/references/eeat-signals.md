# E-E-A-T Implementation Guide

Google's quality raters evaluate content on four dimensions. Each maps to concrete signals you can embed in blog posts.

## Experience

First-hand knowledge demonstrated through real examples.

| Signal                | Implementation                                             |
| --------------------- | ---------------------------------------------------------- |
| Case studies          | "We implemented X and saw Y% improvement"                  |
| Screenshots           | Annotated product/tool screenshots showing real usage      |
| Personal anecdotes    | "When we first encountered this problem..."                |
| Process documentation | Step-by-step with actual commands, configs, outputs        |
| Before/after data     | Concrete metrics from real implementations                 |
| Failure stories       | "Our first attempt failed because..." (builds credibility) |
| Tool comparisons      | Based on actual usage, not feature-list aggregation        |

**Writing pattern**: Use first-person plural ("we tested", "our team found") when describing organizational experience. Use first-person singular for individual expertise.

## Expertise

Demonstrated depth of knowledge and credentials.

| Signal                 | Implementation                                                 |
| ---------------------- | -------------------------------------------------------------- |
| Author bio             | Name, title, credentials, relevant experience, photo           |
| Technical depth        | Go beyond surface-level — explain WHY, not just WHAT           |
| Structured methodology | Explain your analytical framework before presenting results    |
| Terminology precision  | Use correct technical terms; define jargon for mixed audiences |
| Nuanced positions      | Acknowledge trade-offs, not just one-sided recommendations     |
| Original analysis      | Primary research, unique data, novel frameworks                |
| Citation of standards  | Reference RFCs, specifications, official documentation         |

**Writing pattern**: Lead with the principle, follow with the specific implementation. Expert content teaches, not just instructs.

## Authoritativeness

Recognition and reputation within the topic space.

| Signal              | Implementation                                            |
| ------------------- | --------------------------------------------------------- |
| Topical consistency | Blog covers related topics deeply (topical authority)     |
| Internal linking    | Dense internal links showing comprehensive topic coverage |
| Brand mentions      | Cited by others, mentioned in industry publications       |
| Backlink profile    | Links from authoritative sites in your niche              |
| Published research  | Original studies, surveys, data reports                   |
| Speaking/media      | Conference talks, podcast appearances, media quotes       |
| Community presence  | Active in relevant forums, open-source contributions      |

**Content strategy**: Build topical clusters — pillar pages with supporting posts — rather than scattered one-off articles on unrelated topics.

## Trustworthiness

Accuracy, transparency, and reliability signals.

| Signal              | Implementation                                               |
| ------------------- | ------------------------------------------------------------ |
| HTTPS               | Required — no exceptions                                     |
| Contact information | Real company, real address, real team                        |
| Citation sources    | Link to primary sources for every factual claim              |
| Date accuracy       | Published date and last-updated date visible                 |
| Correction policy   | Update posts when information changes; note corrections      |
| Disclosure          | Mark sponsored content, affiliate links, AI-assisted writing |
| Privacy/terms       | Clear privacy policy, terms of service                       |
| Methodology         | Explain how data was collected, analysis performed           |
| Factual accuracy    | Every statistic traceable to a verifiable source             |

**Writing pattern**: "According to [Source] (Year)..." or "[Statistic] ([Source Link])". Never present unverifiable claims as facts.

## E-E-A-T by Content Type

### Technical Tutorial

- **Experience**: Show real terminal output, actual config files
- **Expertise**: Explain edge cases, common pitfalls, why defaults exist
- **Authority**: Link to official docs, reference specifications
- **Trust**: Tested commands/code, version numbers, last-verified date

### Industry Analysis

- **Experience**: "In our work with [N] clients..." or cite survey data
- **Expertise**: Framework for analysis, methodology section
- **Authority**: Cite industry reports, reference recognized benchmarks
- **Trust**: Data sources listed, limitations acknowledged

### Product Comparison

- **Experience**: Hands-on testing evidence (screenshots, benchmarks)
- **Expertise**: Evaluation criteria explained upfront
- **Authority**: Transparent scoring methodology
- **Trust**: Disclose relationships, note when testing was conducted

### How-To Guide

- **Experience**: Photos/screenshots of each step
- **Expertise**: Explain why each step matters, not just what to do
- **Authority**: Reference manufacturer/official guidelines
- **Trust**: Safety warnings, prerequisite checks, troubleshooting section

## AI Content Disclosure

Google's stance: AI-generated content is acceptable IF it provides value. The key signals:

- Human editorial review and fact-checking
- Original insights beyond what AI could synthesize
- Author takes responsibility (byline, bio)
- Content updated based on real-world feedback
- Disclose AI assistance where appropriate

**Pattern**: Use AI for research synthesis and drafting. Add human experience, original data, and editorial judgment before publishing.
